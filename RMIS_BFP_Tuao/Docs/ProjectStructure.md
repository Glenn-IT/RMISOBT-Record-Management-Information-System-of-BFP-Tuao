# Record Management Information System Ś BFP Tuao

## Overview

This project is a WinForms prototype built in VB.NET.
It uses NO database. All data is stored temporarily in memory using List(Of T).
The structure is kept simple and flat Ś no MVC, no layers beyond basic organization.

## ?? Forms/
Contains all WinForms in the system.
Each form has two files generated by Visual Studio:
- `FormName.vb`       ? Code-behind (button clicks, logic)
- `FormName.Designer.vb` ? Auto-generated Designer layout (DO NOT edit manually)

### Forms List:
| Form | Purpose |
|---|---|
| LoginForm.vb | Entry point Ś username/password UI |
| DashboardForm.vb | Main hub Ś shows summary cards and sidebar navigation |
| AddRecordForm.vb | Form to input a new fire incident record |
| ViewRecordsForm.vb | Displays all records in a DataGridView with search |
| ReportsForm.vb | Simple summary/report view |
| SettingsForm.vb | App settings (theme, user preferences) |
| DevelopersForm.vb | Credits Ś developer names and info |

## ?? Models/
Contains plain VB.NET classes with Public Properties only.
No methods, no logic Ś just data shape definitions.

### Files:
| File | Purpose |
|---|---|
| RecordModel.vb | Represents a fire incident record |
| PersonnelModel.vb | Represents a BFP personnel entry |

### Example:
```vb
Public Class RecordModel
    Public Property RecordID As Integer
    Public Property IncidentType As String
    Public Property DateReported As Date
    Public Property Location As String
    Public Property ReportedBy As String
    Public Property Status As String
End Class
```

---

## ?? Services/
Contains the in-memory data service.
Acts as a temporary "database" using List(Of T).
No actual database connection Ś all data is lost when the app closes.

### Files:
| File | Purpose |
|---|---|
| RecordService.vb | CRUD operations on List(Of RecordModel) |

### Methods:
| Method | Description |
|---|---|
| AddRecord() | Adds a new RecordModel to the List |
| GetRecords() | Returns the full List |
| UpdateRecord() | Finds and updates an existing record by ID |
| DeleteRecord() | Removes a record from the List by ID |

## How Forms, Models, and Services Work Together

```
[WinForm UI]
     |
     | (user clicks "Add Record")
     v
[AddRecordForm.vb]
     |
     | (calls)
     v
[RecordService.vb]  ??  [List(Of RecordModel)]
     |
     | (returns data)
     v
[ViewRecordsForm.vb ? DataGridView]
```

1. The **Form** collects input from the user (TextBoxes, ComboBoxes, etc.)
2. It creates a **Model** object and fills its properties
3. It passes the Model to the **Service** which stores it in a List
4. Another Form retrieves the List from the Service and displays it in a DataGridView
